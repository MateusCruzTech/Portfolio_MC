01 · Visão Geral

O que é o Projeto

Este projeto é um aplicativo mobile desenvolvido com Flutter e Dart que implementa um contador com incremento e decremento dinâmico, focado em demonstrar os fundamentos do desenvolvimento mobile com o framework do Google.

O objetivo principal foi entender na prática como o Flutter organiza a interface através da árvore de widgets, a diferença entre widgets Stateless e Stateful, e como o mecanismo de setState() propaga mudanças de estado para a UI de forma reativa e eficiente.

Apesar de ser um projeto de estudo com escopo objetivo, ele cobre conceitos que estão na base de qualquer aplicação Flutter mais complexa · tornando-o um ponto de partida sólido para projetos mobile futuros.

02 · Conceitos Aplicados

Widgets e Gerenciamento de Estado

O Flutter organiza toda a interface como uma árvore de widgets imutáveis. Para que a UI reaja a mudanças · como o valor atual do contador · é necessário trabalhar com widgets Stateful, que mantêm um objeto de estado separado e podem ser reconstruídos quando esse estado muda.

Stateless vs. Stateful

StatelessWidget: usado para componentes que não mudam após serem construídos · como textos estáticos, ícones e layouts fixos. São mais leves e previsíveis, pois não carregam estado interno.

StatefulWidget: usado para componentes que precisam se atualizar em resposta a interações do usuário. No projeto, o contador é implementado como um StatefulWidget · cada clique nos botões chama setState(), que notifica o Flutter para reconstruir apenas a parte afetada da árvore de widgets.

Hierarquia de Componentes

  • Scaffold: estrutura base da tela, com AppBar e área de conteúdo principal.
  • Column: organiza os elementos verticalmente · título, valor do contador e botões de ação.
  • Text: exibe o valor atual do contador, atualizado a cada chamada de setState().
  • ElevatedButton / FloatingActionButton: disparam as ações de incremento e decremento com validação de limite mínimo.
03 · Funcionalidades

O que o App Faz

  • Incremento controlado: adiciona uma unidade ao contador a cada toque, atualizando o valor na tela em tempo real via setState().
  • Decremento com limite mínimo: subtrai uma unidade do contador, impedindo que o valor desça abaixo de zero · lógica de validação implementada antes da chamada ao setState().
  • Atualização reativa da UI: a interface reflete imediatamente qualquer mudança de estado, sem necessidade de recarregar a tela ou navegar entre páginas.
  • Interface responsiva: layout adaptável a diferentes tamanhos de tela usando componentes do Material Design.
04 · Como Executar

Rodando o Projeto Localmente

Para executar o projeto é necessário ter o Flutter SDK instalado e um dispositivo físico conectado ou um emulador Android/iOS configurado. 

# Clone o repositório
git clone https://github.com/MateusCruzTech/App_contador_pessoas.git

# Instale as dependências
flutter pub get

# Execute (dispositivo conectado ou emulador ativo)
flutter run
05 · Reflexões

O que aprendi com este projeto

Trabalhar com Flutter pela primeira vez deixou clara a filosofia do framework: tudo é widget, e a UI é apenas o reflexo do estado atual da aplicação. Entender esse modelo mental · onde você descreve como a tela deve parecer dado um estado, e não como ela deve mudar · foi o aprendizado mais importante do projeto.

O mecanismo de setState(), embora simples, introduz o conceito fundamental de reatividade que está na base de abordagens mais avançadas como Provider, Riverpod e BLoC · tornando este projeto um ponto de partida essencial antes de avançar para gerenciamento de estado mais complexo.

A experiência também despertou interesse em aprofundar o desenvolvimento mobile com Flutter, explorando navegação entre telas, consumo de APIs e persistência de dados em projetos futuros.
